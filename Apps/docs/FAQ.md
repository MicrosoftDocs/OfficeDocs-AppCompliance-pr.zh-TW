## <a name="data-handling"></a>資料處理
|問題                                       |展開的定義                          |

|您的應用程式會收集或處理哪些資料？|提供應用程式所處理的特定資料類型，例如使用者設定檔資料、使用者郵件資料等。|
|應用程式是否支援 TLS 1.1 或更新版本？|傳輸層安全性 (TLS) 1.1 或更新版本是透過電腦網路建立加密通道的安全性通訊協定。 使用 TLS 有助於防止竊聽和攔截式攻擊。 若要為客戶提供最佳的類別加密，您的應用程式是否支援 TLS 1.1 或更新版本？|
|應用程式或基礎結構是否會儲存任何 Microsoft 客戶資料？|如果您的應用程式會處理或儲存任何 Microsoft 客戶資料，例如從 Microsoft Graph 或客戶租使用者等 Microsoft 資源端點取用的任何資料，請選取 [是]。 如果沒有，請選取 [否]。|
|哪些資料會儲存在您的資料庫中？|提供儲存在資料庫中的特定資料類型，例如使用者設定檔資料、使用者郵件資料、租使用者資訊，例如租使用者識別碼、使用者通訊識別碼等。|
|如果基礎結構會處理或儲存 Microsoft 客戶資料，則此資料會在何處儲存？|指定將儲存 Microsoft 客戶資料的區域。 例如： 德國、日本。|
|您是否有已建立的資料租用和處置程式？|當客戶要求刪除資料或取消訂閱時，您的組織是否遵循嚴格的資料儲存或刪除資料標準？|
|帳戶終止之後，您要維護使用者資料多久？|修補客戶離開服務後儲存客戶資訊的時間週期。|
|您是否已建立管理客戶資料、加密金鑰/秘密存取權的已建立程式？|加密是安全性不可或缺的工具，因為它會限制存取，金鑰保存庫讓應用程式和使用者儲存和使用數種類型的秘密/金鑰資料。 是否已處理及收集、分析和管理客戶資料、加密金鑰/秘密的所有存取？ 這項資訊與資料處理和安全性做法有關，因此需要這項資訊。|
|應用程式是否會將任何 Microsoft 客戶資料或客戶內容傳送給協力廠商或子處理者？|客戶資料可以是員工使用者名稱或識別碼、位置、人員資訊、使用者特定 IP 位址等。如果您的組織將任何 Microsoft 客戶資料或客戶內容傳輸給協力廠商或子處理者，請選取 [是]。 如果沒有，請選取 [否]。|
|您是否與共享 Microsoft 客戶資料的任何協力廠商服務有資料共用協定？|協力廠商服務可以是話務中心、BPO、資料輸入等。如果您有與任何協力廠商服務共用 Microsoft 客戶資料的合約，請選取 [是]。 如果沒有，請選取 [否]。|

## <a name="compliance"></a>合規性
|問題                                       |展開的定義                          |
|----------------------------------------       |---------------------------------------------|

|應用程式是否符合健康情況資訊信任聯盟、Common Security Framework (HITRUST CSF) ？HITRUST CSF 是一組控制項，可統合資訊安全性法規和標準的需求。|對於與醫療保健服務相關聯的應用程式，或提供服務給 helthcare 服務，這是必要的。|
|應用程式是否符合服務組織控制 (SOC 1) ？ 報表服務組織中與使用者實體對財務報告的內部控制相關的控制項。| 對於具有與金融服務相關之應用程式或提供服務給金融機構的美國和非美國公司而言，這是必要的。|
|應用程式是否符合服務組織控制 (SOC 2) ？|根據安全性、隱私權、可用性、機密性和處理完整性的一或多個信任服務準則，報告非財務處理。[瞭解更多資訊](https://docs.microsoft.com/compliance/regulatory/offering-home)|
|您達成哪一個 SOC 2 認證？|選取 [類型 1] 或 [類型 2]，如果您已取得這兩個類型，請選取 [類型 2]|
|應用程式是否符合服務組織控制 (SOC 3) ？|根據信任服務準則報告，可自由散發，且只包含管理的判斷提示，指出其符合所選準則的需求？ [深入了解](https://docs.microsoft.com/compliance/regulatory/offering-home)|

|應用程式國際標準組織 (ISO 27001) 認證嗎？|ISO 27001 是一種憑證，提供給支援國際認可指導方針的公司，以及在組織內起始、實作及改善資訊安全性管理的一般原則。[瞭解更多資訊](https://docs.microsoft.com/compliance/regulatory/offering-ISO-27001)|
|應用程式是否符合國際標準組織 (ISO 27018) ？|ISO 27018 會建立常用的控制項，並提供在公用雲端運算環境中處理和保護個人識別資訊 (PII) 的指導方針？[瞭解更多資訊](https://docs.microsoft.com/compliance/regulatory/offering-ISO-27018)|
|應用程式是否符合國際標準組織 (ISO 27017) ？|ISO 27017 會建立常接受的控制項和指導方針，以在公用雲端運算環境中處理和保護使用者資訊。[瞭解更多資訊](https://docs.microsoft.com/compliance/regulatory/offering-ISO-27017)|
|應用程式是否符合國際標準組織 (ISO 27002) ？|ISO 27002 會建立組織資訊安全性標準和資訊安全性管理實務的一般指導方針。[瞭解更多資訊](https://www.itgovernance.co.uk/blog/understanding-the-differences-between-iso-27001-and-iso-27002)|
